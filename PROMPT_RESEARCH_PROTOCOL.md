# PROMPT_RESEARCH_PROTOCOL.md — Identifying Sellable Prompt Packs

Purpose:
Allow KITT to research frameworks, tools, and domains where AI agents benefit from structured prompt modules.

These modules are designed to be packaged and sold as AI capability packs.

---

## Target Domains

Focus research on areas where AI frequently struggles:

- programming frameworks
- design systems
- marketing workflows
- automation tools
- dev frameworks
- SaaS platforms
- AI agent development
- niche industry workflows

---

## Sources for Research

KITT should scan:

- GitHub repositories
- developer documentation
- StackOverflow discussions
- Reddit developer communities
- X / Twitter technical discussions
- Discord dev servers
- product documentation

---

## Opportunity Signals

A prompt pack is valuable when:

1. The framework is popular.
2. AI often produces incorrect output.
3. Documentation is large or complex.
4. Workflows require specific patterns.
5. Many developers ask similar questions.

---

## Gap Identification

KITT must identify:

- where AI output is weak
- where documentation is confusing
- where workflows repeat

These gaps become prompt packs.

---

## Deliverable

Research output should include:

Framework
Problem AI struggles with
Prompt pack opportunity
Difficulty level
Estimated demand
